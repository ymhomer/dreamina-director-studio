# KB-06｜人物与数字人稳定库

> 用途：本知识库用于帮助「即梦导演 Prompt Studio」在处理人物、@角色、数字人、分身、双人互动、多人同框、参考图人物、口型对嘴、舞蹈、变装、剧情表演时，提升人物身份一致性、脸部稳定性、服装发型连续性、动作可控性和画面可生成性。

> 调用场景：当用户使用 @角色、@分身、数字人、AI分身、男女主、双人互动、多人同框、参考图人物、人物一致、脸跑偏、脸不稳定、角色不像、人物变形、对嘴、跳舞、变装、剧情连续、拍续集时，应优先调用本库。

> 本库只负责人物与数字人稳定，不负责平台基础规则、Prompt整体结构、爆款创意、运镜、风格、热点话题和全链路发布。相关内容应分别调用 KB-01、KB-02、KB-03、KB-04、KB-05、KB-07、KB-08。

## 1. 知识库定位

人物稳定是 AI 视频生成中最容易出问题的环节之一。

本库的核心作用是帮助 GPT 判断：

1. 用户是否使用 @角色 / @分身 / 数字人。
2. 用户是否只是使用参考图作为造型或风格参考。
3. 人物身份、服装、发型、动作和镜头是否冲突。
4. 单人、双人、多人画面如何降低跑脸风险。
5. 人物动作是否过快、过复杂或遮挡脸部。
6. 角色是否需要在拍同款、拍续集、系列视频中保持一致。
7. 当用户反馈人物不像、脸崩、动作乱时，如何修正 Prompt。

核心原则：

```text
人物稳定的关键不是描述更多，而是减少冲突、明确身份、降低动作复杂度。
```

## 2. 人物稳定总流程

```mermaid
flowchart TD
    A[用户需求] --> B{是否有人物?}
    B -->|否| C[无需调用人物稳定]
    B -->|是| D{人物来源是什么?}
    D -->|@角色/@分身| E[锁定人物身份]
    D -->|参考图人物| F[判断参考用途]
    D -->|文字原创人物| G[建立人物描述]
    E --> H[判断人数]
    F --> H
    G --> H
    H --> I[设计位置关系]
    I --> J[控制动作复杂度]
    J --> K[选择稳定镜头]
    K --> L[加入人物稳定约束]
    L --> M[输出Prompt]
```

正确顺序：

```text
人物来源 → 人物身份 → 人数关系 → 动作复杂度 → 镜头稳定 → 约束补强
```

错误顺序：

```text
先写大量造型和风格，再希望人物自动稳定
```

## 3. 人物来源分类

人物来源不同，Prompt 写法不同。

| 人物来源 | 含义 | 主要任务 |
|---|---|---|
| @角色 / @分身 | 平台内已存在的数字人或固定角色 | 锁定人物身份与脸部一致性 |
| 参考图人物 | 用户提供图片作为视觉参考 | 参考造型、服装、姿势、风格，不等同固定身份 |
| 文字原创人物 | 完全由文字生成的新人物 | 建立清晰但不过载的人物描述 |
| 群体人物 | 多人或背景角色 | 明确主次，避免抢主体 |
| 产品/非人角色拟人 | 动物、怪兽、物品有拟人行为 | 明确主体形态和行为逻辑 |

## 4. @角色 / @分身调用规则

### 4.1 @角色的本质

@角色用于指定「谁是主角」。

当 Prompt 中出现 @角色时，GPT 应优先保证：

```text
人物身份一致 > 服装风格 > 动作复杂度 > 特效强度
```

@角色负责：

- 人物身份
- 面部稳定
- 系列连续性
- 固定角色记忆
- 人物关系

### 4.2 单人 @角色写法

标准结构：

```text
@角色A作为主角，保持面部、发型与身份一致，穿{服装}，位于{画面位置}，做{简单连续动作}，镜头{稳定景别}，整体{风格}。
```

示例：

```text
@角色A作为主角，保持面部、发型与身份一致，穿简洁蓝色西装，站在舞台中央，面对镜头自信演讲。镜头中景缓慢推进到近景，背景干净，舞台灯光柔和稳定。
```

### 4.3 双人 @角色写法

双人场景必须明确：

- 谁在左
- 谁在右
- 谁先动
- 谁后反应
- 谁是视觉中心
- 两人是否互动
- 动作是否重叠

标准结构：

```text
@角色A在画面左侧，@角色B在画面右侧，两人保持面部清晰。@角色A先{动作}，@角色B随后{反应}，动作不重叠，镜头稳定中景拍摄。
```

示例：

```text
@角色A站在画面左侧，@角色B站在画面右侧，两人面对镜头轮流对嘴演唱，动作同步但不重叠。镜头稳定中景，保持两人面部清晰、服装一致、无多余人物。
```

### 4.4 多人 @角色写法

多人画面必须设置视觉主角。

标准结构：

```text
@角色A为主角站在画面中央，@角色B和@角色C分别位于左右两侧作为互动角色，三人位置固定，动作简洁同步，镜头优先跟随@角色A。
```

### 4.5 @角色稳定约束

当 Prompt 出现 @角色时，应自动加入：

```text
保持@角色面部一致，发型和服装一致，主体清晰居中，动作连贯缓慢，脸部无遮挡，无面部变形，无多余人物，无穿模。
```

如果是双人：

```text
两人位置固定，左右关系清楚，动作不重叠，镜头稳定中景，保持两人脸部清晰。
```

如果是多人：

```text
主角优先清晰，其他人物作为陪衬，队形固定，不做复杂交叉走位。
```

## 5. 参考图人物逻辑

### 5.1 参考图人物的本质

参考图人物主要用于视觉参考，不等同于固定数字人身份。

参考图人物可用于：

- 造型参考
- 服装参考
- 发型参考
- 姿势参考
- 妆造参考
- 气质参考
- 首帧构图参考
- 光影与风格参考

### 5.2 参考图人物标准写法

```text
参考图作为人物造型、服装和发型参考，保持整体气质一致，但动作和场景重新生成。
```

```text
参考图作为首帧构图参考，人物从参考图相似姿势开始，随后缓慢抬头看向镜头。
```

```text
参考图作为妆造和光影参考，保留柔和光线、服装色彩和整体氛围。
```

### 5.3 参考图 + @角色组合

当参考图和 @角色同时出现时，必须明确分工：

```text
@角色负责人物身份。
参考图负责造型、服装、场景、光影或构图参考。
```

组合模板：

```text
@角色A作为主角，保持本人面部与身份一致；参考图作为服装、发型和光影风格参考。主角位于画面中央，动作缓慢自然，镜头稳定中景。
```

场景参考模板：

```text
@角色B作为主角，保持面部稳定；参考图作为场景与光影参考，人物身份以@角色B为准，动作和剧情重新生成。
```

### 5.4 冲突优先级

| 冲突内容 | 优先级 |
|---|---|
| 人物身份 / 脸部 | @角色优先 |
| 服装 / 妆造 | 用户明确指定优先，其次参考图 |
| 场景 / 光影 | 用户指定或参考图优先 |
| 动作 / 剧情 | 用户文字指令优先 |
| 风格 / 调色 | 用户指定风格优先，其次参考图 |

### 5.5 常见错误

错误：

```text
让@角色A完全变成参考图里的人。
```

问题：身份混乱，容易跑脸。

更稳：

```text
@角色A保持本人面部和身份一致，参考图只作为服装、发型和光影参考。
```

错误：

```text
参考图同时作为人物、动作、场景、风格、首帧、尾帧全部参考。
```

问题：参考任务过多，模型难以理解。

更稳：

```text
参考图主要作为首帧构图和服装参考，后续动作重新生成。
```

### 5.6 图片素材中的造型与妆容拆分规则

当用户要求「图片素材」「素材参考图」「服装、发型、妆容与配饰展示图」时，参考图人物逻辑必须从“生成肖像”切换为“生成造型信息板”。

允许呈现：

- 无脸人台、半身假人、背影发型、假发展示、平铺服装、服装设计稿
- 单独眼影色块、单眼眼妆示意、眼线形状、眉形线稿、唇色卡、腮红位置图、高光材质、亮片材质
- 鞋子、手套、饰品、布料、剪裁、纹理和材质特写

禁止呈现：

- 清晰完整人脸
- 完整五官组合
- 真人脸部特写
- 正面肖像照
- 可识别身份的人物脸部
- 将眼睛、鼻子、嘴巴和脸型拼成完整脸

稳定写法：

```text
造型参考图使用无脸人台、平铺服装和分离式妆容设计板展示；妆容元素分散呈现，不组成完整脸；重点是服装、发型轮廓、配饰、材质和色彩，不生成真人肖像。
```

当图片素材与 @角色同时出现时：

```text
@角色负责人物身份；素材图只负责背景、造型、妆容元素、动作或镜头参考，不作为人脸身份来源。
```

## 6. 文字原创人物稳定规则

当用户没有 @角色，也没有参考图时，需要用文字建立原创人物。

### 6.1 原创人物描述结构

```text
身份 + 年龄感 + 造型 + 服装 + 情绪状态 + 画面位置
```

示例：

```text
一名年轻程序员，穿格子衬衫，眼神疲惫但认真，坐在老式木桌前，画面中景居中。
```

```text
一名古代侠客，黑色长发束起，穿深色布衣与皮革护腕，表情冷静，站在客栈门口。
```

### 6.2 原创人物描述原则

1. 不要堆太多外貌细节。
2. 优先描述身份、服装、状态和动作。
3. 年龄感用宽泛描述，不用过度精确。
4. 不要同时要求多个强烈冲突造型。
5. 服装颜色和材质要前后一致。
6. 场景要服务人物，不要抢主体。

### 6.3 稳定原创人物模板

```text
一名{身份}，穿{服装颜色/材质}，{发型/气质}，位于画面中央，动作缓慢自然，镜头中景稳定拍摄，保持外观一致。
```

## 7. 单人稳定策略

单人视频是最稳定的人物类型。

### 7.1 推荐配置

| 项目 | 建议 |
|---|---|
| 景别 | 中景 / 中近景 |
| 动作 | 1条主动作链 |
| 镜头 | 慢推 / 固定机位 / 轻跟拍 |
| 风格 | 1个主风格 |
| 特效 | 不遮挡五官 |
| 时长 | 默认15秒；人物复杂或测试版可缩短为8–12秒 |

### 7.2 单人稳定模板

```text
单人主角位于画面中央，镜头稳定中景拍摄，动作缓慢连贯，脸部始终清晰无遮挡，服装发型保持一致，背景简洁不抢主体。
```

### 7.3 适合动作

```text
缓慢抬头
轻微转身
向前走一步
伸手触碰物体
整理衣领
看向镜头
递出道具
缓慢挥手
对嘴演唱
简单手势舞
```

### 7.4 不适合动作

```text
连续翻滚
高速打斗
快速转圈
大幅度跳跃
脸部被长时间遮挡
强光覆盖五官
快速多角度切脸
```

## 8. 双人互动稳定策略

双人视频比单人更容易出问题，需要明确关系和位置。

### 8.1 双人关系类型

| 类型 | 写法重点 |
|---|---|
| 男女主互动 | 左右位置、眼神、动作顺序 |
| 对唱 MV | 轮流对嘴、动作同步但不重叠 |
| 情侣 POV | 一方靠近镜头，另一方作为视角或对象 |
| 对峙剧情 | 越肩视角、正反打、位置固定 |
| 搞笑反应 | 一人做动作，另一人表情反应 |
| 广告展示 | 一人使用产品，一人观察或回应 |

### 8.2 双人稳定模板

```text
@角色A在画面左侧，@角色B在画面右侧，两人位置固定，脸部清晰。@角色A先做{动作}，@角色B随后做{反应}，动作不重叠，镜头稳定中景拍摄。
```

### 8.3 双人 MV 模板

```text
@角色A与@角色B并肩站在舞台中央，两人轮流对嘴演唱，动作简洁同步，镜头在中景舞蹈和近景对嘴之间切换，保持两人脸部清晰和位置稳定。
```

### 8.4 双人剧情模板

```text
两人面对面站立，@角色A为主视角，@角色B作为互动对象。镜头先拍@角色A中景，再切到@角色B反应近景，最后回到双人中景，位置关系保持一致。
```

### 8.5 双人常见问题

| 问题 | 原因 | 修正 |
|---|---|---|
| 两人脸混 | 位置不明确 | 写清左右位置和服装差异 |
| 动作重叠 | 两人同时复杂动作 | 轮流动作，分先后 |
| 主体不清 | 两人权重相同 | 指定视觉中心 |
| 互动混乱 | 关系没写清 | 写清谁看谁、谁递给谁 |
| 镜头乱 | 快速切换太多 | 用稳定中景为主 |

## 9. 多人同框稳定策略

多人场景风险更高，必须控制队形和主次。

### 9.1 多人场景适合情况

- 群舞
- 直播间背景
- 武林大会
- 工厂劳动场景
- 课堂
- 发布会观众
- 怪兽城市人群

### 9.2 多人稳定原则

```text
一个主角，其他人是陪衬。
```

必须明确：

- 主角位置
- 其他人位置
- 队形
- 主角动作
- 背景人物动作
- 镜头关注对象

### 9.3 多人模板

```text
主角站在画面中央，其他人物分布在左右两侧作为背景陪衬，队形固定。镜头优先跟随主角，背景人物只做简单反应，不抢主体。
```

### 9.4 群舞模板

```text
三人小队站成三角队形，中间主角为视觉中心，左右两人做同步手势，动作简洁，不交叉走位。镜头中景稳定拍摄，副歌处切主角近景。
```

### 9.5 多人避坑

避免：

```text
多人快速跑动
多人互相遮挡
多人复杂打斗
多人同时对嘴
多人近景拥挤
多人快速环绕镜头
```

推荐：

```text
固定队形
中景构图
主角居中
背景轻微虚化
动作分层
镜头优先主角
```

## 10. 人物动作复杂度控制

动作复杂度决定人物稳定性。

## 10.1 动作风险等级

| 风险等级 | 动作类型 | 建议 |
|---|---|---|
| 低风险 | 抬头、转身、走一步、看镜头、递物 | 推荐 |
| 中风险 | 简单舞蹈、对嘴、跑动、轻打斗 | 需简化 |
| 高风险 | 翻滚、快拳、跳跃、多人交叉、快速换装 | 尽量避免 |
| 极高风险 | 多人快打 + 快速环绕 + 强特效 | 不建议 |

## 10.2 稳定动作链

```text
停顿 → 抬头 → 轻微转身 → 看向镜头 → 定格
```

```text
走入画面 → 拿起道具 → 观察 → 表情变化 → 结尾定格
```

```text
对嘴一句歌词 → 简单手势 → 灯光变化 → 眼神定格
```

## 10.3 动作简化原则

复杂动作改为：

```text
高速打斗 → 慢动作蓄力 + 定格出招
复杂舞蹈 → 简洁手势舞 + 节拍点头
快速转圈 → 半身缓慢转身
多人混战 → 双人对峙 + 一个关键动作
连续跳跃 → 一次跳起 + 慢动作落地
```

## 11. 镜头与人物稳定关系

人物越重要，镜头越要稳定。

### 11.1 推荐镜头

| 场景 | 推荐镜头 |
|---|---|
| 单人出镜 | 中景慢推 |
| 双人互动 | 稳定中景 / 正反打 |
| 口播演讲 | 平视中景慢推 |
| 对嘴 MV | 中景 + 近景切换 |
| 变装 | 固定机位 + 遮挡转场 |
| 情绪片 | 中近景慢推 / 侧面柔光 |
| 广告人物 | 中景使用产品 + 产品特写 |

### 11.2 高风险镜头

```text
快速环绕脸部
强手持晃动近景
频繁角度切换
俯拍仰拍快速切换
脸部极近特写加快速动作
多人物快速横移
```

### 11.3 稳定镜头模板

```text
镜头稳定中景拍摄，主体始终清晰居中，关键时刻再缓慢推进到近景。
```

```text
镜头轻微跟随主角移动，保持脸部清晰，不快速旋转。
```

## 12. 服装、发型与妆造一致性

人物跑偏经常不是脸本身问题，而是服装和发型不断变化。

### 12.1 服装描述规则

写清：

```text
颜色 + 类型 + 材质 + 风格
```

示例：

```text
深蓝色西装外套，白色衬衫，简洁商务风。
```

```text
黑色长风衣，金属扣件，冷酷赛博风。
```

```text
白色宽袖古风长袍，轻薄丝绸质感。
```

### 12.2 发型描述规则

```text
短黑发，干净利落。
```

```text
长黑发半束起，发丝自然垂落。
```

```text
柔顺长发，随风轻微飘动。
```

### 12.3 服装稳定约束

```text
全程保持同一套服装、发型和妆造，不随机变化。
```

```text
变装前后只有指定造型发生变化，变装后保持新造型一致。
```

## 13. 变装人物稳定规则

变装是高风险但高吸引力类型。

### 13.1 变装结构

```text
低状态前造型 → 触发动作 → 遮挡/闪白 → 高光后造型 → 定格
```

### 13.2 变装稳定模板

```text
主角开场为{变装前造型}，动作缓慢自然。通过{遮挡/闪白/转身}完成转场，变装后为{变装后造型}，保持同一人物面部一致，服装发型稳定，镜头慢推展示。
```

### 13.3 变装前后必须明确

| 项目 | 变装前 | 变装后 |
|---|---|---|
| 服装 | 普通/低调/邋遢 | 华丽/古风/舞台/赛博 |
| 发型 | 简单/凌乱 | 整理后/长发/舞台造型 |
| 光色 | 平淡 | 强光/舞台/电影感 |
| 情绪 | 疲惫/普通 | 自信/冷峻/惊艳 |
| 镜头 | 固定中景 | 慢推/轻环绕/特写 |

### 13.4 变装避坑

避免：

```text
连续多次变装
多人同时变装
变装时脸部长期遮挡
变装后马上快速跳舞
变装前后风格完全无逻辑
```

推荐：

```text
一次明确变装
一个触发动作
一个高光造型
一个结尾定格
```

## 14. 对嘴与口播稳定规则

### 14.1 对嘴适合场景

- MV
- 直播
- 演讲
- 台词短句
- 角色独白
- K-pop / J-pop / C-pop 表演

### 14.2 对嘴写法

```text
主角面对镜头对嘴演唱 “{短歌词}”，口型自然，表情与歌词情绪同步。
```

```text
主角面对观众说 “{短台词}”，口型清晰，手势简洁，镜头稳定中景。
```

### 14.3 对嘴稳定原则

1. 台词不要太长。
2. 镜头不要快速移动。
3. 脸部不要被遮挡。
4. 表情与台词情绪一致。
5. 双人对嘴要轮流，不要同时抢镜。
6. 对嘴时动作要简洁。

### 14.4 对嘴风险修正

如果口型不稳：

```text
减少台词长度，改为一句短句。
```

```text
使用中近景稳定镜头，人物正面面对镜头。
```

```text
减少手势和身体移动，优先保证口型和脸部清晰。
```

## 15. 舞蹈与动作表演稳定规则

### 15.1 舞蹈风险

舞蹈容易出现：

- 肢体变形
- 动作不同步
- 脸部漂移
- 多人错位
- 服装穿模
- 镜头混乱

### 15.2 稳定舞蹈写法

```text
动作简洁有力，以手势和上半身动作为主，脚步移动少，镜头中景稳定拍摄，动作与鼓点同步。
```

```text
三人保持固定队形，中间主角为视觉中心，左右两人做同步手势，不交叉走位。
```

### 15.3 舞蹈建议

| 场景 | 建议 |
|---|---|
| 单人舞 | 简洁手势 + 中景稳定 |
| 双人舞 | 并肩站位 + 动作同步但不贴太近 |
| 三人舞 | 三角队形 + 主角居中 |
| 群舞 | 背景舞者虚化，主角清晰 |
| K-pop | 中景舞蹈 + 近景对嘴快切 |

### 15.4 不建议

```text
复杂地板动作
多人交叉换位
高速旋转
连续跳跃
镜头快速环绕舞者
极近特写跳舞
```

## 16. 拍续集与系列人物一致性

### 16.1 续集人物规则

续集必须保留：

```text
角色身份
核心外观
服装或主视觉符号
人物关系
上一集状态
主风格
```

### 16.2 续集写法

```text
延续上一集设定，@角色A仍作为主角，保持相同面部、发型和核心服装风格。本集从上一集结尾状态继续，新增{新冲突}，但人物身份和视觉风格不变。
```

### 16.3 系列角色卡

建立系列时，应固定：

```text
角色名：
身份：
外观标志：
常用服装：
性格反应：
固定动作：
关系对象：
世界观规则：
```

### 16.4 系列模板

```text
@角色A固定作为主角，保持面部和身份一致，每集进入不同场景，但保留{固定服装/道具/性格反应}。每集只新增一个新冲突，结尾留下下一集伏笔。
```

## 17. 常见人物失败原因与修正

## 17.1 脸跑偏

原因：

- 风格词过强
- 参考图和 @角色身份冲突
- 动作太快
- 脸部被遮挡
- 近景镜头运动太复杂

修正：

```text
强化@角色身份，减少风格负荷，使用中景稳定镜头，动作缓慢，脸部无遮挡。
```

## 17.2 人物变形

原因：

- 快速动作
- 多人交叉
- 强特效覆盖身体
- 镜头快速旋转

修正：

```text
减少动作数量，改为慢动作展示，固定队形，特效不遮挡人物身体。
```

## 17.3 两个人混脸

原因：

- 位置不清
- 服装太相似
- 动作同时发生
- 镜头切换混乱

修正：

```text
明确左右位置，服装颜色区分，轮流动作，镜头稳定中景。
```

## 17.4 服装乱变

原因：

- 服装描述不明确
- 风格过多
- 变装逻辑没有写清

修正：

```text
明确服装颜色、类型和材质，全程保持同一套服装；如需变装，只变化一次并写清前后造型。
```

## 17.5 对嘴不稳

原因：

- 台词太长
- 脸部不正对镜头
- 动作太多
- 镜头移动太快

修正：

```text
缩短台词，正面中近景，动作简洁，镜头稳定。
```

## 18. 人物稳定约束片段库

### 18.1 通用人物稳定

```text
主体始终清晰居中，面部无遮挡，动作缓慢连贯，服装发型一致，无面部变形，无多余人物。
```

### 18.2 @角色稳定

```text
保持@角色面部和身份一致，全程发型服装稳定，镜头中景为主，避免快速遮挡脸部。
```

### 18.3 参考图人物稳定

```text
参考图只作为造型、服装和光影参考，人物动作和场景重新生成，保持整体气质一致。
```

### 18.4 双人稳定

```text
两人左右位置固定，脸部清晰，动作轮流发生，不重叠，镜头稳定中景拍摄。
```

### 18.5 多人稳定

```text
主角为视觉中心，其他人物作为背景陪衬，队形固定，不做复杂交叉走位。
```

### 18.6 变装稳定

```text
只进行一次明确变装，变装前后造型清楚，人物面部保持一致，转场使用遮挡或闪白完成。
```

### 18.7 对嘴稳定

```text
主角正面面对镜头，对嘴一句短台词，口型自然，动作简洁，镜头稳定。
```

## 19. 人物稳定质量检查清单

输出人物相关 Prompt 前必须检查：

```text
[ ] 是否明确人物来源？
[ ] 如果有@角色，是否锁定人物身份？
[ ] 如果有参考图，是否写清参考用途？
[ ] 如果是图片素材需求，是否避免完整人脸，并将妆容拆成分离式元素？
[ ] 是否明确主角是谁？
[ ] 是否控制人物数量？
[ ] 双人是否写清左右位置？
[ ] 多人是否写清主次？
[ ] 动作是否过快或过复杂？
[ ] 镜头是否过度环绕或频繁切换？
[ ] 脸部是否可能被遮挡？
[ ] 服装和发型是否一致？
[ ] 风格是否压过人物？
[ ] 特效是否遮挡五官？
[ ] 是否加入人物稳定约束？
```

## 20. 本库给 GPT 的执行指令

当调用本库时，GPT 应遵守：

1. 出现 @角色 / @分身 时，优先保证人物身份一致。
2. 参考图人物不等于数字人身份，必须写清参考用途。
3. 图片素材图只负责背景、造型、妆容、动作或镜头参考，不负责锁定人物身份；妆容元素可分离出现，但不能组成完整脸。
3. 单人视频优先中景、慢推、动作简洁。
4. 双人视频必须写清左右位置和动作顺序。
5. 多人视频必须设置主角，其他人为陪衬。
6. 人物越重要，镜头越要稳定。
7. 风格越强，动作越要简单。
8. 动作越复杂，镜头越要稳定。
9. 特效不能遮挡人物五官。
10. 变装只做一次明确变化，前后造型要写清。
11. 对嘴台词要短，镜头要正面稳定。
12. 舞蹈以简洁手势和中景为主，避免复杂走位。
13. 拍续集必须保留人物身份、关系和核心视觉符号。
14. 用户反馈人物不像时，优先减少风格和动作负荷，而不是继续堆外貌词。
15. 输出最终 Prompt 前必须加入必要的人物稳定约束。

## 21. 总结

本库的核心价值是让人物在 AI 视频里更稳定、更清楚、更连续。

最重要的人物稳定公式：

```text
身份清楚 + 位置清楚 + 动作简单 + 镜头稳定 + 风格不过载 = 人物更稳定
```

最重要的优先级：

```text
人物身份 > 脸部清晰 > 动作连贯 > 服装一致 > 风格强化 > 特效表现
```

最终目标：

```text
让即梦 Prompt 里的角色不是“出现一下”，而是能被观众认出、记住，并在下一条视频中继续成立。
```
