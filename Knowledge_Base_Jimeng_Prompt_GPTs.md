# 即梦视频提示词 GPTs 知识库（重构版）

## 0. 知识库定位
这份知识库不是原研究报告的照搬，而是为了 GPTs 实战调用而重构成：
- 可检索
- 可拼装
- 可快速回答
- 可直接产出 Prompt

核心原则：
1. 案例库给灵感
2. 风格库给气质
3. 模板库给结构
4. 原子片段库给细节
5. 规范库给边界
6. 选项库给快速互动
7. 迭代库给优化方法

---

## 1. 即梦向核心规律

### 1.1 平台创作规律
- 优先自然语言，而不是堆叠关键词
- 正向描述比负向限制更稳
- 动作越慢、越连贯，通常越稳
- 一支片尽量只有一条主线
- 镜头语言写清楚，比模糊说法更可控
- 多镜头适合用“2秒钩子 + 中段推进 + 结尾落点”
- 风格要成套表达，不要只丢一个标签

### 1.2 实用默认建议
- 竖屏：9:16
- 横屏：16:9
- 通用时长：6–12 秒
- 剧情/广告/多镜头可延伸到 12–15 秒
- 快速测试先用短版，再做高质量终稿
- 字幕尽量预留安全区，真正的硬字幕更建议后期做

## 2. Prompt 核心模块

标准模块：
1. 场景
2. 主体
3. 动作
4. 情绪
5. 镜头
6. 光色
7. 音效/氛围
8. 后期质感
9. 约束
10. 输出建议

标准骨架：

```text
【场景】时间 / 地点 / 环境 / 道具
【主体】身份 / 外观 / 穿着 / 年龄感
【动作】核心动作A → 过渡 → 核心动作B
【情绪】主情绪 + 情绪变化
【镜头】景别 / 机位 / 运镜 / 节奏
【光色】光线方向 / 主辅色 / 氛围
【音效】环境音 + 同步音 + 音乐调性
【后期】胶片 / 数码 / 暗角 / 雾化 / 颗粒
【约束】主体一致 / 稳定 / 无杂物 / 无多余人物
【参数】比例 / 时长 / 输出建议
```

## 3. 创作类型库

### 3.1 剧情类
特征：
- 有目标
- 有阻碍
- 有转折
- 有结尾落点

适合结构：
- 0–2s：钩子
- 2–6s：推进
- 6–10s：冲突细节
- 10–12/15s：转折或收束

### 3.2 娱乐搞怪类
特征：
- 前2秒就要抓人
- 反差强
- 反转快
- 主体尽量简单

适合结构：
- 开场直接冲突
- 中段升级
- 结尾反转

### 3.3 广告类
特征：
- 前段小剧情 / 小动作
- 后段必须落到产品或核心卖点
- 背景干净
- 文字安全区明确

### 3.4 视觉奇观类
特征：
- 重点不在剧情，在变化过程
- 适合材质、能量、变形、空间折叠
- 强调节奏阶段与材质可见性

### 3.5 艺术情绪类
特征：
- 重氛围、呼吸感、停顿
- 重光影、色调、动作细节
- 情绪可大于情节

## 4. 高复用结构规律

### 4.1 叙事压缩规律
在短时长里最稳的结构是：
- 钩子
- 推进
- 落点

### 4.2 运动稳定规律
- 慢动作比乱动稳
- 连续动作比跳变稳
- 一镜一事比多事同发稳

### 4.3 镜头外显规律
写清楚：
- 推
- 拉
- 摇
- 移
- 跟拍
- 环绕
- 景别
- 机位
会比“有电影感”“镜头动一下”更有效

## 6. 模板库

### 6.1 通用模板
```text
【场景】{时间/地点/环境}，{天气/空间质感/关键道具}
【主体】{角色身份 + 外观特征 + 服装材质 + 年龄感}
【动作】{核心动作A → 过渡 → 核心动作B}
【情绪】{情绪词}，{情绪变化}
【镜头】{景别}，{机位}，{运镜}，{节奏}
【光色】{光源方向}，{主色/辅色}，{氛围}
【音效】{环境音} + {关键同步音} + {BGM调性}
【后期】{质感}
【约束】画面稳定、主体一致、无多余人物、无多余杂物
【参数】{比例}，{时长}，{输出建议}
```

### 6.2 剧情模板
```text
【故事一句话】{主题}：{人物目标}，但{阻碍}，于是{转折/解决}
镜头1（0-2s）：环境建立 + 钩子信息
镜头2（2-6s）：主体动作推进
镜头3（6-10s）：关键冲突或特写
镜头4（10-12/15s）：转折落点或收束
音效：环境音 + 同步音 + 旁白/字幕位
```

### 6.3 娱乐搞怪模板
```text
0-2s：直接给反常识画面或冲突
2-6s：主体认真做一件离谱的事
6-8/10s：画外介入或道具反转
结尾：1秒定格 + 记忆点
```

### 6.4 视觉奇观模板
```text
【奇观对象】{液态金属/能量体/巨大生物/空间折叠}
【变化规则】{变形 → 聚合 → 爆发 → 回收}
【镜头】远景建立 → 中景跟随 → 特写材质
【光色】{轮廓光/体积雾/主辅色}
【节奏】每2-3秒一个阶段
【后期】电影质感 或 干净数码（二选一）
```

### 6.5 艺术情绪模板
```text
【主题】{孤独/治愈/怀旧/狂欢}
【主体】{舞者/行人/乐手/剪影人物}
【动作】重复但有细微变化
【镜头】跟随 / 环绕 / 定镜长镜
【光色】单一主色调 + 少量点睛色
【音效】呼吸 / 脚步 / 布料 / 环境氛围
【后期】颗粒 / 暗角 / 烟雾
```

## 7. 原子片段库（可插拔）

### 7.1 叙事与节奏
- 开场就是冲突：画面里出现不合常理的动作
- 先给结果，再倒回原因
- 动作分三段：开始 → 意外 → 修正
- 每2秒出现一个新的信息点，但主线动作不变
- 听到画外脚步声，角色立刻停住
- 一道闪光掠过，场景无缝切换
- 最后1秒定格，留字幕安全区
- 镜头缓慢拉远，情绪逐渐冷却
- 慢、连、稳：动作缓慢、过渡连贯、画面稳定
- 避免同时发生多件事：一个主体 + 一个核心动作
- 关键动作瞬间音效必须对齐画面
- 情绪曲线：紧张 → 稳住 → 高潮 → 余韵

### 7.2 镜头
- 远景建立环境尺度
- 中景表现人物动作
- 近景或特写强调情绪与细节
- 低机位跟拍，速度缓慢
- 镜头缓慢推进，强调主观引导
- 镜头缓慢拉远，用于收场
- 轻微环绕半圈，保持稳定
- 水平平移扫过，像翻页一样衔接
- 焦点在雨滴与武器细节之间切换
- 主体居中，背景简化，留字幕安全区
- 负空间明显，主体只占画面三分之一
- 闪白 + 运动匹配转场，尽量无抖动

### 7.3 光色与质感
- 侧逆光，轮廓光勾边
- 柔光散射，皮肤细节自然
- 顶光产品特写，背景纯黑
- 黄高光 + 青绿阴影，略欠曝
- 粉紫蓝主调，深色城市底
- 红光点睛，其余画面压暗
- 16mm胶片颗粒，轻微暗角
- VHS扫描线与轻微色偏
- 布料褶皱与重力感自然
- 金属碰撞瞬间有清晰高频响声
- 画面稳定无抖动，镜头丝滑
- 主体外观一致，面部清晰不变形
- 背景不要出现无关人物与杂物
- 避免矛盾指令：不要同时要求极慢动作与极快运镜

## 9. 平台规范与建议值

### 9.1 已知方向
- 支持多模态参考
- 长时长可到 15 秒级别
- 多镜头输出能力强
- 有素材时更容易控画面
- 运镜、音效、构图都可作为描述重点

### 9.2 实战建议值
- 比例：9:16 / 16:9 / 1:1
- 时长：6–12 秒最通用
- 多镜头剧情：12–15 秒
- 测试：短时长先试
- 字幕：预留位置，成片再后期更稳
- 有人物参考时注意授权

### 9.3 风险提醒
- 真人人像需授权
- 公众人物、品牌、IP 不建议直接硬描摹
- 不要把多个冲突风格强塞一条 Prompt
- 高复杂动态 + 多角色 + 多特效同时发生，容易崩

## 11. 回答策略库（给 GPT 内部调用）

### 11.1 当用户很模糊
先帮他缩小范围，不直接乱生成：
- 先问方向
- 再问镜头
- 再问风格
- 最后问字幕/声音

### 11.2 当用户很清楚
直接给：
- 最终 Prompt
- 必要时再补一个更强导演版

### 11.3 当用户只回复字母
自动识别并合并，例如：
- A + C = 剧情 + 广告质感
- B + D + “港风一点” = 搞怪 + 预留字幕 + 港风

### 11.4 当用户想跳出选项
直接吸收，不纠正，不要求重选

## 12. 迭代优化工作流

1. 明确目标
2. 选母版
3. 选风格包
4. 选镜头节奏
5. 组装 Prompt
6. 先做短版测试
7. 看问题是：
   - 动作太乱
   - 主体不稳
   - 风格太杂
   - 镜头不清
   - 文字太挤
8. 再修正到终稿

### 12.1 A/B 测试建议
只改一个变量：
- 结构 A/B
- 镜头 A/B
- 风格 A/B

不要一次全改，不然不知道哪部分有效。

## 13. 可直接作为系统判断的优先级

高优先级：
- 主体明确
- 主线明确
- 动作可执行
- 镜头可执行
- 风格统一
- 无冲突

中优先级：
- 细节丰富
- 声音氛围
- 字幕提示
- 材质质感

低优先级：
- 冗长修辞
- 炫技但无助于生成的形容词
